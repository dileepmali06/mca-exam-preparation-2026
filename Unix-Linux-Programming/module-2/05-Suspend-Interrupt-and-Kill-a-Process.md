# Introduction

Linux is a multitasking operating system that allows multiple processes to execute simultaneously.

Sometimes a process may:

- Run for a long time.
- Stop responding.
- Consume excessive CPU or memory.
- Execute an incorrect command.
- Need to be paused temporarily.

To handle these situations, Linux provides process control operations such as:

- Interrupt
- Suspend
- Resume
- Kill

These operations help users manage running processes efficiently.

---

# Why Do We Need Process Control?

Consider the following situations:

- A command is running forever.
- A program enters an infinite loop.
- A process stops responding.
- A process is consuming excessive system resources.

Instead of restarting the entire system, Linux allows us to control individual processes.

---

# Process Control Operations

Linux provides three major operations:

```text
Running Process
      │
      ├── Interrupt
      ├── Suspend
      └── Kill
```

---

# 1. Interrupt a Process

## Definition

Interrupt means stopping a running foreground process immediately.

Linux generates an **Interrupt Signal (SIGINT)** when the user presses:

```text
Ctrl + C
```

This requests the running process to terminate.

---

# Example

Run:

```bash
ping google.com
```

Output

```text
64 bytes from ...
64 bytes from ...
64 bytes from ...
```

Press:

```text
Ctrl + C
```

The process stops immediately and the terminal becomes available.

---

# SIGINT

- Signal Name: **SIGINT**
- Signal Number: **2**
- Generated By: `Ctrl + C`
- Purpose: Interrupt the running process.

---

# 2. Suspend a Process

## Definition

Suspend means temporarily stopping a running process without terminating it.

The process remains in memory and can be resumed later.

---

# Shortcut

```text
Ctrl + Z
```

---

# Example

Run:

```bash
sleep 100
```

Now press:

```text
Ctrl + Z
```

Output

```text
[1]+ Stopped sleep 100
```

The process is now suspended.

---

# Resume a Suspended Process

## Resume in Background

```bash
bg
```

The suspended process continues in the background.

---

## Resume in Foreground

```bash
fg
```

The process resumes in the foreground and occupies the terminal again.

---

# Suspend Process Flow

```text
Running

↓

Ctrl + Z

↓

Suspended

↓

bg

↓

Background

↓

fg

↓

Foreground
```

---

# 3. Kill a Process

## Definition

Kill means permanently terminating a process.

Unlike Suspend, a killed process cannot continue execution.

---

# kill Command

## Syntax

```bash
kill PID
```

Example

```bash
kill 4521
```

The process with Process ID 4521 will be terminated.

---

# Finding the Process ID (PID)

Before killing a process, we need its PID.

Commands:

```bash
ps
```

or

```bash
ps -ef
```

or

```bash
top
```

Example Output

```text
PID TTY TIME CMD

4521 pts/0 sleep
```

---

# killall Command

The `killall` command terminates all processes having the same name.

Example

```bash
killall firefox
```

All Firefox processes are terminated.

---

# pkill Command

The `pkill` command terminates processes based on their name or pattern.

Example

```bash
pkill sleep
```

All `sleep` processes are terminated.

---

# Linux Signals

Linux communicates with running processes using **Signals**.

Signals are software interrupts sent to processes.

---

# Common Signals

| Signal | Number | Purpose |
|---------|--------|----------|
| SIGINT | 2 | Interrupt Process |
| SIGTERM | 15 | Gracefully Terminate Process |
| SIGKILL | 9 | Forcefully Kill Process |

---

# SIGINT

Generated using:

```text
Ctrl + C
```

Purpose:

Interrupt the running process.

---

# SIGTERM

Generated using:

```bash
kill PID
```

It politely requests the process to terminate.

The process gets an opportunity to release resources before exiting.

---

# SIGKILL

Generated using:

```bash
kill -9 PID
```

This forcefully terminates the process.

The process cannot ignore this signal.

---

# kill vs kill -9

| kill | kill -9 |
|------|----------|
| Sends SIGTERM | Sends SIGKILL |
| Graceful termination | Forceful termination |
| Allows cleanup | No cleanup |
| Recommended first | Use only if necessary |

---

# kill vs killall vs pkill

| Command | Works On | Example |
|----------|----------|---------|
| `kill` | Process ID (PID) | `kill 4521` |
| `killall` | Process Name | `killall firefox` |
| `pkill` | Process Name or Pattern | `pkill sleep` |

# Common Beginner Mistakes

### Mistake 1

Thinking `Ctrl + C` and `Ctrl + Z` are the same.

**Correct:**

- `Ctrl + C` → Interrupts the process.
- `Ctrl + Z` → Suspends the process.

---

### Mistake 2

Using `kill -9` every time.

**Correct:**

Always try:

```bash
kill PID
```

first.

Use `kill -9` only when the process does not terminate.

---

### Mistake 3

Using Job Number instead of PID with `kill`.

**Correct:**

The `kill` command requires the **Process ID (PID)**.

# Summary

- Linux provides process control through Interrupt, Suspend, and Kill operations.
- `Ctrl + C` sends **SIGINT** to interrupt a running process.
- `Ctrl + Z` suspends a running process without terminating it.
- `bg` resumes a suspended process in the background, while `fg` brings it back to the foreground.
- `kill` sends **SIGTERM**, `kill -9` sends **SIGKILL**, `killall` terminates processes by name, and `pkill` terminates processes using a name or pattern.
- These commands are essential for Linux system administration, development, and process management.
