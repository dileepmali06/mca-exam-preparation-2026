# 5.3.2 Microsoft Azure

## Introduction to Microsoft Azure

Microsoft Azure is a cloud computing platform developed by Microsoft that provides a wide variety of cloud services without requiring organizations to purchase and maintain physical hardware. Azure enables developers and businesses to build, test, deploy, and manage applications through Microsoft-managed data centers.

Azure offers services such as compute, storage, networking, databases, security, analytics, artificial intelligence, and application development tools.

### What is Azure?

Microsoft Azure is a collection of cloud computing services that helps organizations host existing applications, develop new applications, and manage cloud infrastructure efficiently. Azure supports scalability, high availability, security, and global deployment.

### Benefits of Azure

* No need to purchase physical servers.
* Pay-as-you-go pricing model.
* Easy scalability and flexibility.
* Global availability through multiple regions.
* Strong security and compliance support.
* Integration with Microsoft products and services.

---

# Azure Service Categories

Azure services are broadly divided into four major categories:

## 1. Compute Services

Compute services provide processing power required to run applications in the cloud.

### Azure Virtual Machines (VM)

Virtual servers that allow users to run operating systems and applications in Azure.

### Azure Websites

Service used to host web applications and websites.

### Azure Mobile Services

Backend support for mobile applications.

### Purpose of Compute Services

* Run applications.
* Host websites.
* Execute business logic.
* Process user requests.

---

## 2. Data Services

Data services are used for storing and managing cloud data.

### Azure Storage

Provides scalable cloud storage for files, images, videos, and documents.

### Azure SQL Database

Managed relational database service.

### Redis Cache

Provides fast in-memory caching for frequently accessed data.

### Purpose of Data Services

* Store structured and unstructured data.
* Improve application performance.
* Provide scalable storage solutions.

---

## 3. Application Services

Application services help build and operate cloud applications.

### Azure Active Directory (Azure AD)

Provides authentication, authorization, and identity management.

### Azure Service Bus

Enables communication between distributed applications.

### HDInsight

Big-data processing platform based on Hadoop and Spark.

### Azure Scheduler

Used to schedule and automate recurring tasks.

### Azure Media Services

Provides video processing, streaming, and media management capabilities.

### Purpose of Application Services

* Identity management.
* Messaging and integration.
* Big-data processing.
* Workflow automation.
* Media streaming.

---

## 4. Network Services

Network services connect cloud resources and users securely.

### Virtual Network (VNet)

Creates isolated private networks within Azure.

### Azure Content Delivery Network (CDN)

Delivers content quickly to users worldwide.

### Azure Traffic Manager

Distributes user traffic across multiple resources and regions.

### Purpose of Network Services

* Secure connectivity.
* Traffic distribution.
* Performance optimization.
* Global content delivery.

---

# How Azure Works

Azure is built on virtualization technology and operates through a hierarchical architecture consisting of data centers, clusters, racks, and servers.

## Rack

A rack contains multiple servers along with power supplies and networking equipment.

## Fabric Controller

The Fabric Controller is responsible for monitoring and managing servers inside a rack.

Functions:

* Monitors server health.
* Detects failures.
* Performs recovery operations.
* Allocates resources.

### Fabric Controller = Rack Manager

---

## Orchestrator

The Orchestrator acts as the central controller of Azure resources.

Functions:

* Create resources.
* Update resources.
* Delete resources.
* Manage requests through APIs.

### Orchestrator = Azure Resource Manager

---

## Azure Request Processing

When a user requests a resource through Azure Portal or PowerShell:

### Step 1: Authentication

Azure verifies the identity of the user.

### Step 2: Authorization

Azure checks whether the user has permission to perform the requested action.

### Step 3: Resource Availability Check

Azure checks available CPU, RAM, storage, and other resources.

### Step 4: Resource Allocation

The request is sent to the appropriate Fabric Controller and servers.

### Step 5: Resource Creation

The requested resource is created and made available to the user.

# Azure Global Infrastructure

Azure operates globally through multiple layers.

## Geography

A geography is a large market area containing one or more Azure regions.

Examples:

* India
* United States
* Europe

Purpose:

* Data residency
* Legal compliance
* Regulatory requirements

---

## Azure Region

A region is a collection of data centers deployed within a specific geographical boundary.

Examples:

* Central India
* South India
* East US
* West Europe

Purpose:

* High performance
* Low latency
* Regional availability

---

## Availability Zones

Availability Zones are physically separate locations within an Azure region.

Benefits:

* Fault tolerance
* Disaster recovery
* High availability

If one zone fails, applications continue running in other zones.

---

## Data Centers

Data centers are physical facilities containing servers, storage systems, networking devices, and cloud infrastructure.

---

# Azure Pricing

Azure follows a Pay-As-You-Go pricing model.

Organizations pay only for the resources they consume.

## Features of Azure Pricing

### Free Credits

Microsoft provides free credits to new users for learning and testing Azure services.

### Pay-As-You-Go

Users are charged according to actual resource consumption.

### Scalability

Resources can be increased or decreased based on business requirements.

### Cost Optimization

Organizations avoid large upfront infrastructure investments.
