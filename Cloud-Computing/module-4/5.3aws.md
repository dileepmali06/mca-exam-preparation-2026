# 5.3 Amazon Web Services (AWS)

## Introduction to AWS

Amazon Web Services (AWS) is Amazon's cloud computing platform that provides flexible, reliable, scalable, and cost-effective cloud solutions. AWS offers Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS).

### History of AWS

* 2002 – AWS Services launched
* 2006 – Cloud products launched
* 2012 – First customer event held
* 2015 – Revenue reached $4.6 Billion
* 2016 – Revenue crossed $10 Billion
* 2016 – Introduced Snowball and Snowmobile
* 2019 – Offered nearly 100 cloud services
* 2021 – AWS provided more than 200 services

---

# Important AWS Services

## 1. Compute Services

### Amazon EC2 (Elastic Compute Cloud)

Virtual machine service that provides complete OS-level control over cloud servers.

### Lightsail

Simple cloud platform for deploying websites and applications quickly.

### Elastic Beanstalk

Platform for automated deployment and management of web applications.

### Amazon EKS

Managed Kubernetes service for deploying and managing containerized applications.

### AWS Lambda

Serverless computing service that runs code without managing servers and charges only when code executes.

---

## 2. Migration Services

### Database Migration Service (DMS)

Used to migrate databases from one platform to another.

### Server Migration Service (SMS)

Helps migrate on-premise servers to AWS.

### Snowball

Physical device used for transferring large amounts of data into and out of AWS.

---

## 3. Storage Services

### Amazon Glacier

Low-cost archival storage for long-term data backup.

### Amazon EBS (Elastic Block Store)

Persistent block storage attached to EC2 instances.

### AWS Storage Gateway

Connects on-premise environments with AWS cloud storage.

---

## 4. Security Services

# Advantages of AWS

* Supports familiar programming languages and databases.
* Pay-as-you-go pricing model.
* No need to maintain physical data centers.
* Fast deployment of infrastructure.
* Easy scalability.
* Virtually unlimited capacity.
* Low Total Cost of Ownership (TCO).
* Centralized billing and management.
* Hybrid cloud support.
* Global deployment across multiple regions.

---

# Disadvantages of AWS

* Advanced support requires paid plans.
* Possibility of downtime and cloud-related issues.
* Default resource limits exist.
* Infrastructure changes may affect application performance.

---

# AWS Best Practices

* Design systems for failure tolerance.
* Decouple application components.
* Keep data close to users and compute resources.
* Balance security and performance.
* Use resources efficiently to reduce cost.
* Prefer reserved instances for long-term workloads.
* Monitor infrastructure continuously.
* Implement backup and disaster recovery strategies.
