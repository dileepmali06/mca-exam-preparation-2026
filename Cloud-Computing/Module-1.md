# Module 1 - Introduction to Cloud Computing

## 1. Cloud Computing

### Definition

Cloud Computing is a technology that provides computing resources such as servers, storage, databases, networking, software, and applications over the Internet on demand.

### Simple Meaning

Cloud Computing me hume apna hardware kharidne ki zarurat nahi hoti. Hum internet ke through resources use karte hain aur jitna use karte hain utna hi pay karte hain.

### Examples

* Gmail
* Google Drive
* AWS
* Microsoft Azure
* Netflix
* ChatGPT

### Benefits

* Cost Reduction
* Scalability
* High Availability
* Flexibility
* Global Access

---

## 2. Web 2.0

### Benefits

---

## 3. Cloud Computing Properties

### Essential Characteristics

#### On-Demand Self Service

Users can access resources whenever needed without human intervention.

#### Broad Network Access

Services can be accessed from anywhere through the Internet.

#### Resource Pooling

Resources are shared among multiple users.

#### Rapid Elasticity

Resources can increase or decrease quickly based on demand.

#### Measured Service

Users pay only for what they use.

---

## 4. Virtualization

### Definition

Virtualization is a technology that creates multiple virtual machines on a single physical machine.

### Example

A physical server can run multiple virtual machines such as:

* Ubuntu VM
* Windows VM
* Linux VM

### Advantages

* Better Resource Utilization
* Cost Reduction
* Scalability
* Isolation
* Easy Management

### Hypervisor

#### Type 1 Hypervisor

Runs directly on hardware.

Examples:

* VMware ESXi
* Hyper-V

#### Type 2 Hypervisor

Runs on top of an operating system.

Examples:

* VirtualBox
* VMware Workstation

---

## 5. Role of Virtualization in Enabling the Cloud

Virtualization is the foundation of Cloud Computing.

### Roles

## 12. Disaster Recovery (DR)

### Definition

Disaster Recovery is the process of restoring systems, applications, and data after a disaster.

### Examples of Disasters

* Fire
* Flood
* Server Failure
* Cyber Attack
* Human Error

### Goals

* Minimize Downtime
* Prevent Data Loss
* Ensure Business Continuity

### Important Terms

#### RPO (Recovery Point Objective)

Maximum acceptable data loss.

#### RTO (Recovery Time Objective)

Maximum acceptable recovery time.

### DR Techniques

* Backup
* Replication
* Failover
* Recovery Planning

---

## 13. Backup and Disaster Recovery in Cloud

### Backup

Creating copies of data.

### Disaster Recovery

Restoring complete systems and services after failure.

### Benefits

* Data Protection
* Fast Recovery
* Reduced Downtime
* Business Continuity

---

## Technologies in Next-Gen Cloud Computing

### 1. Unikernels

A lightweight operating system designed for a single application.

#### Advantages

* Fast Boot Time
* Small Size
* Better Security
* High Performance
* Less Resource Usage

---

### 2. Blockchain

A distributed digital ledger technology that stores data in linked blocks across multiple computers.

#### Features

* Decentralization
* Transparency
* Security
* No Single Point of Failure

#### Example

Bitcoin (2009)

---

### 3. Container-as-a-Service (CaaS)

A cloud service model used to deploy and manage containers.

#### Examples

* Docker
* Kubernetes
* AWS ECS
* AWS EKS

#### Benefits

* Faster Deployment
* Scalability
* Portability
* Resource Efficiency

---

### 4. Software Defined Networking (SDN)

A networking architecture that allows centralized control of networks using software.

#### Advantages

* Centralized Management
* Automation
* Scalability
* Better Security
* Reduced Cost
