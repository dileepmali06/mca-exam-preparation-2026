# Why Do We Need Max Pooling?

After Convolution, feature maps are still large.

Problems:

-   High memory usage
-   Slow computation
-   More training time
-   Risk of overfitting

Solution:

**Max Pooling** reduces the feature map size while keeping the most
important information.

------------------------------------------------------------------------

# What is Max Pooling?

**Definition**

Max Pooling is a downsampling technique that selects the maximum value
from a small region of the feature map.

Simple Meaning:

It keeps only the strongest feature from every small block.

------------------------------------------------------------------------

# Working of Max Pooling

Example Feature Map:

``` text
1 3 2 4
5 6 1 2
7 8 3 1
2 5 9 4
```

Using a **2×2** pooling window:

``` text
1 3
5 6
```

Maximum = **6**

Next block:

``` text
2 4
1 2
```

Maximum = **4**

Next block:

``` text
7 8
2 5
```

Maximum = **8**

Next block:

``` text
3 1
9 4
```

Maximum = **9**

Final Output:

``` text
6 4
8 9
```

# Complete Flow

``` text

      │
      ▼

```

------------------------------------------------------------------------

# Pool Size

Pool Size defines the dimensions of the pooling window.

Common choices:

-   2 × 2 (Most Common)
-   3 × 3

------------------------------------------------------------------------

# Stride

Stride defines how many pixels the pooling window moves after each
operation.

Common value:

-   Stride = 2

------------------------------------------------------------------------

# Max Pooling vs Average Pooling

  Max Pooling                    Average Pooling
  ------------------------------ --------------------------
  Selects maximum value          Selects average value
  Preserves strongest features   Produces smoother output
  Most commonly used             Less commonly used

------------------------------------------------------------------------

# PyTorch Example

``` python
import torch.nn as nn

pool = nn.MaxPool2d(
    kernel_size=2,
    stride=2
)
```

# Limitations

# One-Line Definitions

-   **Pooling:** Reduces the size of a feature map.
-   **Max Pooling:** Selects the maximum value from a pooling window.
-   **Average Pooling:** Uses the average value of the pooling window.
-   **Pool Size:** Size of the pooling window.
-   **Stride:** Distance the pooling window moves.

------------------------------------------------------------------------

# Quick Revision

-   Max Pooling follows Convolution.
-   It reduces spatial dimensions.
-   It keeps only the strongest features.
-   Common Pool Size = 2×2.
-   Common Stride = 2.

# Practice MCQs

## Easy

### Q1. What is the main purpose of Max Pooling?

A. Increase image size

B. Reduce feature map size

C. Increase dataset size

D. Add neurons

**Answer:** B

**Explanation:** Max Pooling reduces feature map size while preserving
important features.

------------------------------------------------------------------------

### Q2. Max Pooling selects:

A. Minimum value

B. Maximum value

C. Average value

D. Random value

**Answer:** B

**Explanation:** It keeps the highest value in each pooling window.

------------------------------------------------------------------------

### Q3. Most common pool size is:

A. 1×1

B. 2×2

C. 5×5

D. 7×7

**Answer:** B

**Explanation:** 2×2 is the standard pooling window.

------------------------------------------------------------------------

### Q4. Pooling is mainly used after:

A. Softmax

B. Convolution

C. Batch Normalization

D. Dropout

**Answer:** B

**Explanation:** Pooling usually follows the convolution layer.

------------------------------------------------------------------------

## Medium

### Q5. Stride controls:

A. Number of filters

B. Window movement

C. Number of neurons

D. Learning rate

**Answer:** B

**Explanation:** Stride determines how far the pooling window moves.

------------------------------------------------------------------------

### Q6. Which PyTorch class performs Max Pooling?

A. nn.Conv2d

B. nn.MaxPool2d

C. nn.Linear

D. nn.BatchNorm2d

**Answer:** B

**Explanation:** nn.MaxPool2d performs 2D max pooling.

------------------------------------------------------------------------

### Q7. Pooling helps reduce:

A. Memory usage

B. Computation

C. Overfitting

D. All of the above

**Answer:** D

**Explanation:** Smaller feature maps reduce computation and memory and
can help reduce overfitting.

------------------------------------------------------------------------

## Advanced

### Q8. Why is Max Pooling preferred over Average Pooling?

A. It preserves strongest activations.

B. It increases image size.

C. It removes all pixels.

D. It replaces convolution.

**Answer:** A

**Explanation:** Strong activations often represent important features.

------------------------------------------------------------------------

### Q9. Which statement is TRUE?

A. Pooling increases spatial dimensions.

B. Pooling creates new pixels.

C. Pooling reduces spatial dimensions.

D. Pooling replaces the input image.

**Answer:** C

**Explanation:** Pooling downsamples the feature map.

------------------------------------------------------------------------

### Q10. Correct CNN order is:

A. Pooling → Convolution

B. Convolution → Max Pooling

C. Softmax → Pooling

D. BatchNorm → Input

**Answer:** B

**Explanation:** A convolution layer is typically followed by a pooling
layer.

------------------------------------------------------------------------

# Final Summary

Max Pooling is a downsampling technique used after convolution to reduce
the size of feature maps while preserving the most important features.
It makes CNNs faster, reduces memory usage, helps reduce overfitting,
and improves computational efficiency.
