# Why Do We Need Weight Initialization?

Before training starts, a neural network has no knowledge.

Therefore, initial values must be assigned to every weight.

This process is called **Weight Initialization**.

Good initialization: - Faster convergence - Stable training - Better
gradient flow

Poor initialization: - Slow learning - Unstable training - Poor accuracy

------------------------------------------------------------------------

# What is Weight Initialization?

**Definition**

Weight Initialization is the process of assigning initial values to the
weights of a neural network before training begins.

# Zero Initialization

All weights are initialized to zero.

Example:

``` text
W1 = 0
W2 = 0
W3 = 0
```

### Why is it Bad?

All neurons produce identical outputs and receive identical gradients.

This is called the **Symmetry Problem**.

❌ Not recommended.

------------------------------------------------------------------------

# Symmetry Problem

``` text
Neuron 1 → Same Output

Neuron 2 → Same Output

Neuron 3 → Same Output
```

All neurons learn the same features.

------------------------------------------------------------------------

# Random Initialization

Weights are assigned random values.

Example:

``` text
W1 = 0.15
W2 = -0.27
W3 = 0.42
```

### Advantages

-   Breaks symmetry.
-   Allows neurons to learn different features.

### Limitation

Very small or very large random values may slow or destabilize training.

------------------------------------------------------------------------

# Xavier (Glorot) Initialization

### Purpose

Designed for Sigmoid and Tanh activation functions.

### Advantages

-   Stable gradients
-   Faster convergence
-   Balanced signal flow

------------------------------------------------------------------------

# He Initialization

### Purpose

Designed specifically for ReLU and Leaky ReLU.

### Advantages

-   Better gradient flow
-   Stable deep network training
-   Faster learning

------------------------------------------------------------------------

# Comparison

  Initialization   Best Used With
  ---------------- -------------------
  Zero             Never recommended
  Random           Small models
  Xavier           Sigmoid, Tanh
  He               ReLU, Leaky ReLU

------------------------------------------------------------------------

# PyTorch Example

``` python
import torch.nn as nn
import torch.nn.init as init

layer = nn.Linear(128, 64)

init.xavier_uniform_(layer.weight)

# OR

init.kaiming_uniform_(layer.weight)
```

# Final Summary

Weight Initialization is the first step of neural network training. It
assigns starting values to weights before learning begins. Zero
Initialization should be avoided due to the Symmetry Problem. Random
Initialization breaks symmetry, while Xavier Initialization is best
suited for Sigmoid/Tanh and He Initialization is preferred for
ReLU-based deep networks.
